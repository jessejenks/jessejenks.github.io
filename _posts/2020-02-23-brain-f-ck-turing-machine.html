The Brainf*ck Language

Brainf*ck (or BF) is one of the most famous "esoteric" programming languages. Invented in 1993 by Urban Müller, it was intentionally designed to be an extremely small language. The language consists of {% include footnote-link id="keywords" content="only 8 keywords" %} meant to correspond to the behavior of a Turing machine. Every program starts on the first cell of an infinite tape of cells, and each cell can hold a value between 0 and 255. You can move the tape "pointer" left or right and increment or decrement the values in a cell. The most difficult part to get your head around is using the brackets ([ and ]) to repeat a block of code over and over again. But it is this ability to conditionally run a block of code which promotes BF from a language which just puts values in cells to a {% include reference-link id="proof-of-turing-completeness" content="full-fledged Turing-complete programming language" %}.

Notes

  • I opted for a "wrapping" implementation of brainf*ck, meaning that if you decrement a cell with a 0 in it, it will wrap around to 255. Similarly, if a cell contains 255, incrementing will cause it to wrap to 0.
  • In most implementations of BF, output is interpreted as the ascii encoding of a character. So for example, to print the character 'A', you need to increment a cell to 65, and then output with the "." instruction. However, I felt this would make even simple programs take far too long to animate, so, by default, output is just the literal number in a cell.
